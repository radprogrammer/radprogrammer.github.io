How licensing works

The license model follows the same philosophy as the formatter itself: predictable, offline, and no surprises.

Never expires

Licenses are perpetual. A build you're entitled to keeps working forever — there is no lapse state, no kill switch, and no date at which your formatter stops.

Never phones home

License validation is entirely local. Neither the plugin nor the CLI makes any network connection — no activation servers, no license phone-home, no telemetry.

One file, all your versions

A single license file encodes every RAD Studio version you're entitled to. Buy more versions later and you get one updated file, not a pile of keys.

A year of updates, then your choice

Every purchase includes updates for one year. After that, nothing changes until you decide a newer build is worth it — no subscription, no auto-renewal.

ProductLicense file location
IDE plugin%APPDATA%\radFormatter\ (per-user, no admin required), with %ProgramData%\radFormatter\ as the machine-wide fallback
CLISearch order: RADFORMATTER_LICENSE environment variable → --license argument → next to the exe → %APPDATA% → %ProgramData%. Environment-variable-first keeps CI configuration painless

If a license file is missing or invalid, nothing breaks mysteriously: the IDE plugin runs as Community Edition and its options screen explains exactly what's wrong with the license it found; the CLI exits nonzero with a diagnostic listing the paths it checked and the precise reason.
